# Tabi 商业企划书

> 版本：v2026.02
> 日期：2026-02-25

---

## 1. 一句话项目定义

Tabi 是一个把旅游视频快速转化为可发布旅行 HandBook 的内容平台，服务对象是内容运营团队、旅游 KOL 和旅游企业客户。

---

## 2. 我们解决什么问题

### 2.1 创作者侧问题

1. 视频内容难直接变现：粉丝看完视频仍需手工整理地点与路线。
2. 二次整理成本高：把视频变成图文指南，往往要花数小时。
3. 缺少旅游垂直工具：现有通用工具不适合“视频 -> 攻略 -> 分发”的完整链路。

### 2.2 粉丝侧问题

1. 信息分散：地点、玩法、路线散落在视频片段中。
2. 行动成本高：无法快速获得一份可直接参考的完整攻略。

### 2.3 企业侧问题

1. 获客内容生产成本高：旅游企业需要大量高质量目的地内容。
2. 缺少可规模化方案：传统人工编辑难以支持高频更新和多渠道分发。

---

## 3. 产品价值（商业视角）

1. 对运营团队：快速批量生产可交付的旅行 HandBook。
2. 对 KOL：把视频内容沉淀为“可售卖/可公开传播”的数字内容资产。
3. 对旅游企业：通过 Token 化采购方式，把内容能力分发给终端旅游用户。

---

## 4. 当前进展（面向投资人）

1. 产品已可用：从视频到 HandBook 的核心流程已跑通，可持续生成内容样本。
2. 用户体系已可用：支持游客、注册用户与第三方登录用户，适合后续商业化。
3. 内容编辑体验已可用：可进行面向普通用户的可视化编辑与发布前调整。
4. 当前阶段定位：从“产品可用”进入“内容规模化与商业验证”阶段。

---

## 5. 关键产品规划：HandBook 三态（待上线）

我们将把 HandBook 管理为三种状态：

1. 草稿（Draft）：内部编辑与迭代，未对外展示。
2. 归档（Archived）：保留历史内容，不参与对外分发。
3. 公开（Public）：对外可见，可被其他用户浏览与消费。

价值：

1. 支撑内部内容运营流程（先做再发）。
2. 支撑 KOL 发布机制（可选择是否公开）。
3. 为后续“公开内容池/内容广场”打基础。

---

## 6. 推广与商业落地路径（你确认的 4 步）

### 第一步：内部内容运营先跑通（当前正在进行）

1. 公司内容运营团队先用工具筛选并解析 YouTube 视频，生成 HandBook。
2. 把这些 HandBook 作为样本给 KOL 看，降低合作沟通成本。
3. 在这个阶段持续优化工具、修复 bug、沉淀流程。

### 第二步：KOL 合作与 C 端验证

1. 与 KOL 合作，让 KOL 用自己的视频生成 HandBook。
2. KOL 是我们的核心 C 端用户，后续可选择将内容公开给粉丝看。
3. “是否由 KOL 直接售卖给粉丝”的模式暂未完全确定，会在合作中验证。

### 第三步：B 端商业化（以 gojapan 为样板）

1. 面向旅游企业客户提供 Token 采购模式。
2. 例如 gojapan 作为企业客户采购 Token，再分发给其旅游用户使用。
3. 这批用户既可使用企业侧内容，也可查看 KOL 公开 HandBook。

### 第四步：公开内容运营与平台化变现

1. 运营“公开 HandBook 内容池”，提升自然流量和用户停留。
2. 通过 C 端内容供给 + B 端分发渠道形成双边网络效应。
3. 在内容规模和使用规模到位后，扩大为平台化收入模型。

---

## 7. 商业模式

### 7.1 C 端（KOL/创作者）

1. 订阅制（Free / Pro / Max）。
2. 按配额、发布能力、品牌能力做分层。
3. 中期可探索 KOL 内容售卖分成（仍在验证阶段）。

### 7.2 B 端（旅游企业）

1. Token 采购：企业预付采购内容使用额度。
2. 企业分发：企业把额度分发给其终端旅游用户。
3. 可扩展到年度框架协议（量大客户）。

### 7.3 平台运营收入（中长期）

1. 公开内容池的流量变现（品牌合作、目的地合作）。
2. 主题内容活动与联合营销。
3. 数据洞察与行业报告（后续扩展项）。

---

## 8. 阶段目标与里程碑（12-18个月）

### 阶段 A：内容与合作验证（0-6 个月）

1. 完成稳定的内部运营内容生产流程。
2. 累积可展示样本库，建立首批 KOL 合作。
3. 上线 HandBook 三态（草稿/归档/公开）。

### 阶段 B：C 端转化验证（6-12 个月）

1. 建立 KOL 持续使用与公开发布机制。
2. 上线基础订阅体系并验证付费转化。
3. 形成可复用的渠道合作打法。

### 阶段 C：B 端规模验证（12-18 个月）

1. 以 gojapan 为样板完成 Token 模式闭环。
2. 拓展更多旅游企业客户。
3. 建立“公开内容运营 + 企业分发”的双引擎增长结构。

---

## 9. 核心经营指标（建议）

1. 月新增 HandBook 数量（供给规模）。
2. 公开 HandBook 占比（可分发内容比例）。
3. KOL 月活与复用率（C 端粘性）。
4. C 端付费转化率（商业化效率）。
5. B 端 Token 采购额与复购率（企业收入质量）。

---

## 10. 风险与应对

1. KOL 付费意愿不及预期  
应对：先以公开发布和分发价值驱动，再逐步引入高级权益。

2. 企业客户决策周期长  
应对：先做样板客户（如 gojapan）+ 标准化交付模板。

3. 内容质量不稳定影响品牌  
应对：内部运营先行，建立“先草稿后公开”的质量审核机制。

4. 商业路径过早分散  
应对：优先级固定为“样本积累 -> KOL 合作 -> B 端 token 化”。

---

## 11. 融资用途建议（种子轮）

1. 40%：内容运营与增长（样本库建设、KOL 合作、渠道拓展）。
2. 35%：商业化能力建设（订阅、权限、发布与运营机制）。
3. 25%：产品稳定性与交付效率提升（保障规模化使用）。

---

## 12. 结论

Tabi 当前最重要的不是“再讲一个技术故事”，而是把你定义的四步路径快速跑通：

1. 内部内容运营形成稳定供给。
2. KOL 成为真实 C 端用户并持续发布。
3. 以 gojapan 为样板落地 B 端 Token 采购。
4. 通过公开内容池形成平台化增长与盈利能力。

这条路径清晰、可执行、可验证，适合进入下一阶段融资沟通。
