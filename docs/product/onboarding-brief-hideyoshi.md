## TL;DR

Tabi 是一个 **AI 驱动的旅游内容转化平台**，核心能力是把 YouTube 旅行视频自动变成精美的旅行指南手册。目标用户是旅游 KOL/红人，帮助他们将视频内容资产化、可分发化。

**当前阶段：** MVP 已完成，核心 AI Pipeline 可用，尚未上线付费体系。

---

## 1. 我们在解决什么问题？

旅游红人拍了大量优质视频，但：
- 视频里的攻略信息（地点、美食、路线）散落在视频中，粉丝很难直接用
- 手动整理成图文攻略要 **5-10 小时**，大部分红人不会去做
- 市场上没有专门做"视频 → 旅游指南"的工具

**我们的定位：** 5 分钟内，一键把视频变成可直接使用的旅行手册。

---

## 2. 产品怎么工作的？

用户操作只需 2 步：
1. 粘贴 YouTube 链接
2. 选一个手册风格

剩下的全部由 AI 自动完成：

```
粘贴链接 → 抓取视频+字幕 → AI 提取地点/美食/交通等
  → 自动地理编码 → 智能配图 → 生成 HTML 旅行手册
```

最终产出一份**带图片、带地图坐标、响应式**的精美旅行指南，可预览、下载、分享。

---

## 3. 当前产品能做什么？（已实现）

| 能力 | 说明 |
|------|------|
| YouTube 视频解析 | 自动抓取视频元数据 + 字幕 |
| AI 内容结构化 | 从字幕中提取地点、美食、交通、购物等结构化信息 |
| 地理编码 | 自动将地点转为经纬度坐标 |
| 双模式配图 | Unsplash 真实照片搜索 + Google Imagen AI 生成 |
| 手册生成 | 5 种视觉风格的响应式 HTML 手册 |
| 内容编辑 | 可编辑 AI 提取的内容块（标题、描述、坐标、标签） |
| 实时进度 | 流式展示 AI 执行过程 |
| 自动保存 | 编辑内容实时保存，支持断点恢复 |

### 5 种手册风格

| 风格 | 调性 |
|------|------|
| Minimal Tokyo | 日式极简，大量留白 |
| Warm Analog | 复古旅行杂志质感 |
| Brutalist | 高对比度，粗体几何 |
| Dreamy Soft | 柔和渐变，空灵优雅 |
| Let Tabi Decide | AI 自动选择 |

## 4. 还缺什么？（Gap）

| 优先级 | 缺失功能 | 为什么重要 |
|--------|----------|-----------|
| **P0** | PDF 导出 | 付费转化的核心交付物 |
| **P0** | Google Maps 一键导入 | 商业计划的核心差异化卖点 |
| **P1** | 用户认证 + 付费体系 | 没有这个就没有收入 |
| **P1** | 拖拽式编辑器（Gamma 风格） | 当前编辑器体验偏基础 |
| **P1** | 多视频合并 | Pro 版核心付费功能 |
| **P2** | B2B API | 第二增长曲线 |
| **P2** | 自定义品牌/模板 | Max 版差异化 |

## 6. 竞争格局

| | Tabi | Gamma | Notion AI | Canva |
|---|---|---|---|---|
| 旅游场景专注 | ✅ | ❌ 通用 | ❌ 通用 | ❌ 通用 |
| 视频解析 | ✅ | ❌ | ❌ | ❌ |
| 地图集成 | ✅ | ❌ | ❌ | ❌ |
| 编辑器 | ⚠️ 基础 | ✅ 强 | ✅ 强 | ✅ 强 |
| 多格式输出 | ⚠️ HTML only | ✅ | ✅ | ✅ |

**核心壁垒：** 旅游垂直深耕 + AI Agent Pipeline + 地图能力，通用工具很难追上。

**核心短板：** 编辑器体验和输出格式多样性，这是通用工具的强项。

## 9. 待讨论的关键问题

以下是产品线发展中需要对齐的核心议题，供讨论参考：

### 产品方向
1. **编辑器投入优先级** — 当前编辑器是基础表单式，要不要投入做 Gamma 级别的拖拽编辑器？投入大但体验提升明显
2. **输出格式扩展** — PDF 导出是 P0，但要不要同时考虑 Notion/小红书/Instagram 等平台格式？
3. **Google Maps 一键导入** — 这是商业计划的核心卖点，技术上坐标已有，主要是产品交互和导入流程设计

### 商业模式
4. **定价策略** — Free 3 次/月是否合理？Pro $19 vs Max $49 的功能分界是否清晰？
5. **B2B 时机** — 商业计划说 Year 2 启动，但核心 Pipeline 已具备 API 化基础，是否可以提前？
6. **变现路径** — 先做订阅还是先做联盟营销（酒店/景点佣金）？哪个更快验证？

### 增长策略
7. **目标市场** — 英语 vs 中文市场优先级？日本旅游内容是一个自然的切入点？
8. **获客渠道** — 红人合作伙伴计划的具体激励设计？Product Hunt 的 timing？
9. **内容护城河** — 用户生成的旅行指南数据能否形成数据飞轮？

### 技术投入
10. **LLM 成本控制** — 当前每次生成涉及多个 API 调用（DeepSeek + Gemini + Imagen/Unsplash），单次成本需要评估
11. **视频源扩展** — 只支持 YouTube 是否足够？抖音/TikTok/B站 的需求优先级？
